## 3. エクスポートした5つの GeoTIFF データを、5バンドを含む1つのマルチバンドラスタデータにまとめる

ここでは、R、G、B、Red Edge、NIR の各バンドとしてエクスポートした5つのGeoTIFFファイルを、ArcGIS Pro上で1つのマルチバンドラスタデータに統合する。

P4 Multispectral で取得した各波長帯のオルソ画像は、はじめはバンドごとに別々のGeoTIFFファイルとして出力される。
しかし、後続の処理では、同じ地点における複数バンドの値をまとめて扱う必要がある。

そのため、個別に出力された B、G、R、Red Edge、NIR の5つのGeoTIFFを、1つのマルチバンドラスタとして結合する。
これにより、1つのラスタデータ内に5つの波長帯情報を保持でき、バンド間の対応関係を保ったまま解析できるようになる。

<br>
<br>
## 入力データ

この処理で使用する入力データは、各バンドごとにエクスポートしたGeoTIFFファイルである。

例：

B.tif
G.tif
R.tif
REDEDGE.tif
NIR.tif

各GeoTIFFは、同じ範囲、同じ解像度、同じ座標系で作成されている必要がある。
位置や解像度が異なる場合、後続のバンド値抽出時に、同一地点の値として正しく対応しない可能性がある。
<br>
<br>
## 使用するツール

ArcGIS Pro の以下のツールを使用する。

Data Management Tools
→ Raster
→ Raster Processing
→ Composite Bands

日本語表示の場合は、以下のツールを使用する。

データ管理ツール
→ ラスタ
→ ラスタ処理
→ コンポジット バンド
3.4. 処理手順

ArcGIS Proで「コンポジット バンド」ツールを開き、入力ラスタとして5つのGeoTIFFファイルを指定する。

入力ラスタは、解析で使用するバンド順に並べる。
本処理では、以下の順番で入力する。

1. B.tif
2. G.tif
3. R.tif
4. REDEDGE.tif
5. NIR.tif

この順番で入力することで、作成されるマルチバンドラスタのバンド構成は以下のようになる。

Band 1 = B
Band 2 = G
Band 3 = R
Band 4 = Red Edge
Band 5 = NIR

出力ラスタ名を指定し、ツールを実行する。

出力例：

Site7_9_CompositeBands

<br>
<br>

## 出力データ

この処理により、5バンドを含む1つのマルチバンドラスタデータが作成される。

作成されたラスタデータには、B、G、R、Red Edge、NIR の各バンド情報が格納されており、以降の調査区での切り抜き処理や、バンド値の抽出処理に使用する。
<br>
<br>
## 注意点

入力するGeoTIFFの順番は非常に重要である。
後続の処理では、Band 1をB、Band 2をG、Band 3をR、Band 4をRed Edge、Band 5をNIRとして扱うため、入力順が異なると、抽出されるバンド値の意味が入れ替わってしまう。

そのため、「コンポジット バンド」を実行する際には、必ず以下の順番で入力する。

B → G → R → Red Edge → NIR

また、5つのGeoTIFFが同じ座標系・解像度・範囲で作成されていることを確認してから処理を行う。
